# GCD 알고리즘 분석

## 1. GCD 알고리즘_유클리드 호제법

```
int gcd(int a, int b) {
    while (b != 0) {
        int temp = a % b;
        a = b;
        b = temp;
    }
    return a;
}
```

## 2.시간 복잡도 

유클리드 호제법은 두 정수 a, b에 대해  
(a, b) → (b, a % b)의 과정을 반복하며 최대공약수를 구한다.

- 각 반복에서 두 번째 값 b는 나머지 연산에 의해 빠르게 감소 
이 과정에서 입력 값이 커질수록 반복 횟수가 증가하지만, 그 증가 속도는 매우 완만

즉, 반복 횟수는 입력 크기에 대해 로그적으로 증가

* 특히 최악의 경우는 두 수가 연속된 피보나치 수일 때 발생,
이 경우 반복 횟수는 피보나치 수열의 성질에 의해 입력 크기에 대해 로그 수준으로 증가한다.

## 3. Big-O 표기
시간 복잡도: O(log n)
- 유클리드 호제법의 반복 횟수는 입력 값 n에 대해 로그 수준으로 증가하므로 시간 복잡도는 O(log n)이다.
시간 복잡도: O(log n)
* 여기서 n은 입력값 a와 b 중 더 큰 값을 의미
